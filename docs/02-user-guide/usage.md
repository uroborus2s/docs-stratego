# 使用说明

本文面向文档维护者，介绍聚合站点的日常操作、发布验证及故障排查。

## 1. 核心任务清单

维护者最常见的任务有：

1. **本地预览**：在提交前验证页面路径和导航。
2. **发布验证**：确保 GitHub Actions 发布成功且权限正确。
3. **子仓联动管理**：审核并合并子仓同步 PR。
4. **版本回滚**：在出现严重问题时快速恢复。

---

## 2. 本地预览

### 目标
验证页面路径、导航和构建结果。

### 操作步骤
本地开发模式（直接读取本机子仓目录）：
```bash
./start.sh --source-mode local
```

本地生产预演（模拟 CI 环境拉取远程子仓）：
```bash
./start.sh --source-mode remote
```

### 成功判断
- 访问 `http://127.0.0.1:8001/`。
- 确认顶部导航和左侧目录显示正常。
- 确认 OpenAPI/MCP 契约已正确渲染。

---

## 3. 子仓同步管理

根仓通过“子仓变更驱动”的机制自动聚合文档。

### 自动同步流程
1. **子仓触发**：子仓 push 文档变更，通知根仓。
2. **根仓 PR**：根仓自动创建名为 `chore: sync source repository pointers` 的共享 PR。
3. **审核与合并**：维护者审核该 PR 的预览效果，确认无误后执行 `Squash and merge`。

### 详细接入指南
若需要接入新的子项目，请参考：**[子仓库接入指南](subrepo-integration.md)**。

---

## 4. 发布与验证

### 自动发布
提交到根仓 `main` 分支后，GitHub Actions 会自动执行 `Deploy Docs` 工作流，将站点发布到生产服务器。

### 权限验证
发布完成后，必须执行以下验证：
1. **公开页访问**：匿名状态下可正常阅读首页及公开文档。
2. **私有页锁定**：点击带有 `access: private` 标记的页面，应触发登录小窗。
3. **登录闭环**：使用 Casdoor/GitHub 账号登录后，可正常阅读私有内容。

---

## 5. 故障处理矩阵

| 现象 | 可能原因 | 处理建议 |
| :--- | :--- | :--- |
| **构建报错：子仓分支不存在** | `source-repos.json` 配置错误 | 核对该文件中的 `branch` 声明。 |
| **私有页面匿名可见** | `docs/index.md` 缺少 `access: private` | 在子仓修正导航配置并重新同步。 |
| **登录小窗不弹出** | `access-control.js` 加载失败 | 检查站点静态资源路径。 |
| **Actions 成功但页面没更新** | 服务器缓存或 Nginx 配置未生效 | 尝试执行 `sudo systemctl reload nginx`。 |
