# 阅读者指南 (Reader Guide)

本文档面向所有需要访问和查看聚合站点的团队成员，介绍站点的访问逻辑与登录流程。

## 1. 站点解决什么问题
`docs-stratego` 聚合了多个子项目的文档。您无需在不同的仓库中寻找 Markdown 文件，而是在这套统一的门户中进行一站式阅读。

## 2. 两类访问方式

### 2.1 匿名访问 (Public Pages)
- **公开页面**可以直接阅读。
- 即使在同一个导航中，公开页面也不会要求您登录。

### 2.2 登录访问 (Private Pages)
- **私有页面**受权限保护。
- 当您点击一个被保护的私有页面时，系统会自动弹出一个**登录小窗**。
- 您可以通过 **GitHub 登录** 或 **本地账号** 进行身份验证。
- **登录完成后，小窗会自动关闭**，主页面将立即加载受保护的内容。

## 3. 常见问题 (FAQ)

### 3.1 为什么有些页面要求登录？
如果页面在导航中可见但点击后触发登录，说明该文档被子项目维护者标记为 `access: private`。这通常包含内部设计细节、API 密钥或敏感文档。

### 3.2 登录小窗没有弹出怎么办？
- 请检查您的浏览器是否拦截了弹出窗口。
- 刷新页面并重试。

### 3.3 我可以直接访问私有页面 URL 吗？
可以。如果您直接输入私有页面的 URL 且未登录，系统会引导您完成登录流程，并在登录成功后自动跳转回目标页面。
