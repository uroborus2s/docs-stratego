# 云服务器部署与 CI/CD 实操

## 1. 目标

本文给出当前推荐的生产落地方式：

- 文档构建：GitHub Actions Runner
- 静态站点目录：`/var/www/docs-stratego`
- 私有规则文件：`/etc/nginx/snippets/docs-stratego/private_locations.conf`
- 认证运行目录：`~/docs-stratego`
- 认证栈：`casdoor + oauth2-proxy + Redis`
- 网关：宿主机 `Nginx`

标准发布链路是：

1. Runner 构建 `site/`
2. Runner 生成 `private_locations.conf`
3. Runner 上传两者到服务器
4. 服务器只负责安装静态站点、安装规则文件、reload `Nginx`

## 2. 服务器最终布局

```text
~/docs-stratego/
└── deploy/
    ├── docker-compose.yml
    ├── casdoor/
    │   ├── app.conf
    │   └── data/
    └── oauth2-proxy/
        └── oauth2-proxy.cfg

/var/www/docs-stratego/
└── ... site files ...

/etc/nginx/sites-available/
├── docs.example.com
└── auth.docs.example.com

/etc/nginx/snippets/docs-stratego/
└── private_locations.conf
```

## 3. 首次服务器准备

### 3.1 稀疏拉取认证运行目录

```bash
git clone --filter=blob:none --no-checkout https://github.com/uroborus2s/stratego-docs.git ~/docs-stratego
cd ~/docs-stratego
git sparse-checkout init --no-cone
git sparse-checkout set /deploy/docker-compose.yml /deploy/casdoor/ /deploy/casdoor/** /deploy/oauth2-proxy/ /deploy/oauth2-proxy/**
git checkout main
mkdir -p ~/docs-stratego/deploy/casdoor/data
```

### 3.2 创建运行时目录

```bash
sudo install -d -m 0755 -o root -g root /var/www/docs-stratego
sudo install -d -m 0755 -o root -g root /etc/nginx/snippets/docs-stratego
printf "# generated by docs-stratego deploy\n" | sudo tee /etc/nginx/snippets/docs-stratego/private_locations.conf >/dev/null
sudo chmod 0644 /etc/nginx/snippets/docs-stratego/private_locations.conf
```

### 3.3 创建 Docker 网络

```bash
docker network create docs-auth-internal
```

不要迁移现有 Redis。  
保持 Redis 继续留在原网络，例如 `webapp_wps_net`，并让 `oauth2-proxy` 同时加入：

- `docs-auth-internal`
- `webapp_wps_net`

## 4. 启动 Casdoor 与 oauth2-proxy

先编辑：

- `~/docs-stratego/deploy/casdoor/app.conf`
- `~/docs-stratego/deploy/oauth2-proxy/oauth2-proxy.cfg`

然后启动：

```bash
cd ~/docs-stratego
export DOCS_INTERNAL_DOCKER_NETWORK=docs-auth-internal
export DOCS_REDIS_DOCKER_NETWORK=webapp_wps_net
docker compose -f deploy/docker-compose.yml up -d
docker compose -f deploy/docker-compose.yml ps
curl -I http://127.0.0.1:8000
curl -I http://127.0.0.1:4180/oauth2/auth
```

## 5. 安装 Nginx 站点文件

### 5.1 `docs.example.com`

`/etc/nginx/sites-available/docs.example.com`：

```nginx
server {
  listen 80;
  server_name docs.example.com;

  root /var/www/docs-stratego;
  index index.html;

  include /etc/nginx/snippets/docs-stratego/private_locations.conf;

  location /oauth2/ {
    proxy_pass http://127.0.0.1:4180;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Scheme $scheme;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_set_header X-Forwarded-Host $host;
    proxy_set_header X-Auth-Request-Redirect $request_uri;
  }

  location / {
    try_files $uri $uri/ $uri.html =404;
  }
}
```

### 5.2 `auth.docs.example.com`

`/etc/nginx/sites-available/auth.docs.example.com`：

```nginx
server {
  listen 80;
  server_name auth.docs.example.com;

  location / {
    proxy_pass http://127.0.0.1:8000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_set_header X-Forwarded-Host $host;
  }
}
```

启用：

```bash
sudo ln -sf /etc/nginx/sites-available/docs.example.com /etc/nginx/sites-enabled/docs.example.com
sudo ln -sf /etc/nginx/sites-available/auth.docs.example.com /etc/nginx/sites-enabled/auth.docs.example.com
sudo nginx -t
sudo systemctl reload nginx
```

## 6. 申请证书并切到 HTTPS

申请证书：

```bash
sudo certbot certonly --nginx -d docs.example.com -d auth.docs.example.com
```

然后把两个站点文件改成 HTTPS 版。
具体正式配置内容直接以 [安装说明](installation.md) 中的两个 HTTPS 示例为准。

## 7. 配置 GitHub App 与 Casdoor

如果你要启用 GitHub 登录，建议使用 GitHub App：

1. 在 GitHub 创建 GitHub App
2. 把 `Client ID` / `Client Secret` 填到 Casdoor 的 GitHub Provider
3. 在 Casdoor Application 中启用该 Provider
4. 保持 `redirect_url=https://docs.example.com/oauth2/callback`

关键配置核对：

```ini
oidc_issuer_url = "https://auth.docs.example.com"
redirect_url = "https://docs.example.com/oauth2/callback"
```

## 8. 配置 GitHub Actions Secrets

在 GitHub 仓库：

`Settings -> Secrets and variables -> Actions`

新增这些 Secrets：

| Secret | 示例 |
| --- | --- |
| `DOCS_DEPLOY_HOST` | `203.0.113.10` |
| `DOCS_DEPLOY_USER` | `deploy` |
| `DOCS_DEPLOY_SSH_KEY` | 部署私钥内容 |
| `DOCS_DEPLOY_PORT` | `22` |
| `DOCS_DEPLOY_SITE_DIR` | `/var/www/docs-stratego` |
| `DOCS_PRIVATE_LOCATIONS_PATH` | `/etc/nginx/snippets/docs-stratego/private_locations.conf` |
| `DOCS_RELOAD_HOST_NGINX` | `1` |

配置方法：

1. 打开 GitHub 仓库的 `Settings -> Secrets and variables -> Actions`
2. 点击 `New repository secret`
3. 把上表 7 个参数逐个录入

部署账号要求：

- `DOCS_DEPLOY_USER` 必须能无密码执行 `sudo install`、`sudo rm`、`sudo mv`、`sudo find`、`sudo nginx -t`、`sudo systemctl reload nginx`

当前 workflow 会先跑：

1. `uv sync --frozen`
2. 单元测试
3. `sync_sources.py`
4. `build_site.py`
5. `mkdocs build`

全部通过后，再执行：

1. 上传 `site/` 到 `/var/www/docs-stratego`
2. 安装 `private_locations.conf` 到 `/etc/nginx/snippets/docs-stratego/private_locations.conf`
3. 执行 `sudo nginx -t && sudo systemctl reload nginx`
4. 将 `site/` 与 `private_locations.conf` 作为 artifact 额外保留 7 天

这里保留 artifact 的主要目的是：

1. 让 `validate` job 的输出传给 `deploy` job
2. 在部署失败时，保留 7 天可下载的构建快照

## 9. 首次验证 CI/CD

在本地确认一个小改动后，提交到 `main`：

```bash
git add .
git commit -m "docs: verify production deployment pipeline"
git push origin main
```

然后在 GitHub Actions 里观察：

1. `validate` job 通过
2. `deploy` job 通过

服务器侧再抽查：

```bash
ls -ld /var/www/docs-stratego
ls -l /etc/nginx/snippets/docs-stratego/private_locations.conf
docker compose -f ~/docs-stratego/deploy/docker-compose.yml ps
sudo nginx -t
```

浏览器侧抽查：

1. 打开 `https://docs.example.com`
2. 打开一个公开页面
3. 打开一个私有页面
4. 测试本地密码登录
5. 测试 GitHub 登录
6. 登录后确认回到原页面

## 10. 常见问题

### 10.1 Actions 成功但页面没更新

- 检查 `DOCS_DEPLOY_SITE_DIR` 是否和 `docs.example.com` 站点文件中的 `root` 一致
- 检查 workflow 的 `Upload site tarball` 和 `Publish site and rules` 步骤是否成功
- 必要时下载本次 Actions artifact，核对其中的 `site/` 与 `private_locations.conf`

### 10.2 私有页没有被拦截

- 检查 `/etc/nginx/sites-available/docs.example.com` 是否 `include /etc/nginx/snippets/docs-stratego/private_locations.conf;`
- 检查 `/etc/nginx/snippets/docs-stratego/private_locations.conf` 是否已被本次发布更新

### 10.3 oauth2-proxy 起不来

- 检查 `oauth2-proxy` 是否已接入 Redis 所在网络
- 检查 `client_id/client_secret`

### 10.4 Casdoor 起不来

- 检查 SQLite 文件权限
- 检查 `~/docs-stratego/deploy/casdoor/app.conf`
