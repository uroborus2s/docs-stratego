# 用户指南可读性评审

## 1. 评审目标

本轮评审从“第一次接手 `docs-stratego` 的阅读者”视角出发，验证 `02-user-guide/` 是否满足以下要求：

1. 能在很短时间内判断“我该读哪一页”
2. 关键操作页能给出明确步骤、成功标志和常见误区
3. 发布 CLI 所需的外部配置步骤清晰可执行
4. 高频操作页包含截图或示意图，降低理解门槛

## 2. 评审场景

### 场景 A：普通阅读者

问题：

- 只想看文档，不知道为什么有些页面会弹出登录小窗

结果：

- [阅读者指南](../../02-user-guide/reader-guide.md) 已补齐“公开页 / 私有页 / 直接访问私有 URL”的行为说明
- 已增加登录小窗示意截图，阅读者可以快速理解这是页面级权限，不是整站异常

### 场景 B：根仓维护者

问题：

- 想快速本地预览文档，但不清楚 `local` 和 `remote` 的区别

结果：

- [本地开发与预览](../../02-user-guide/local-development.md) 已补齐 `local` / `remote` 选择表、watch 边界、成功标志和重启条件
- 已增加本地 watch 模式示意截图

### 场景 C：源仓接入执行者

问题：

- 不清楚命令在哪个仓库执行，也不清楚“校验通过”后是否还需要 remote 构建验证

结果：

- [子仓库接入指南](../../02-user-guide/usage.md) 已重构为任务型入口
- [接入聚合站点](../../02-user-guide/contributor-guide/onboarding.md) 已明确区分源仓侧和根仓侧步骤，并强调 remote 构建验证不可跳过
- 已增加接入流程示意截图

### 场景 D：源仓联动配置执行者

问题：

- 不清楚 `DOCS_STRATEGO_DISPATCH_TOKEN` 配在哪里，也不清楚自动联动是否等于自动发布

结果：

- [自动联动](../../02-user-guide/contributor-guide/automation.md) 已明确 Secret 配在源仓，且自动联动只形成共享 PR，不直接发布
- 已增加通知 workflow 示意截图

### 场景 E：根仓维护者审核共享 PR

问题：

- 不清楚共享 PR 的审核顺序和“不应该直接合并”的判断标准

结果：

- [维护者指南](../../02-user-guide/operator-guide.md) 已补齐共享 PR 审核顺序、发布后闭环验证和排障优先级
- 已增加共享 PR 审核示意截图

### 场景 F：管理员首次搭平台和发布 CLI

问题：

- 不清楚安装文档是否适用于本地开发
- 不清楚发布 CLI 前的 GitHub / TestPyPI / PyPI 外部配置顺序

结果：

- [安装说明](../../02-user-guide/installation.md) 已补齐“这页适合谁”“安装 5 阶段”和部署拓扑示意截图
- [发布前外部配置](../../02-user-guide/contributor-guide/publish-setup.md) 已补齐 GitHub Environments、Trusted Publisher、Actions 手动演练的步骤和示意截图

## 3. 本轮发现并已修复的主要问题

| 问题 | 已采取的修复 |
| --- | --- |
| 首页能列出页面，但不能快速形成任务路径 | 重构 `02-user-guide/index.md`，按任务和角色给出最短阅读顺序 |
| 高操作频率页面缺少视觉辅助 | 为阅读、开发、接入、联动、审核、安装补充示意截图 |
| 接入、联动、发布之间的边界容易混淆 | 重构 `usage.md` 与 `Contributor Guide` 页面关系，按闭环顺序组织 |
| 用户指南首页与导航层级不完全一致 | 调整根导航，使“阅读与访问 / 开发与接入 / 运维与发布 / 平台管理”一致 |

## 4. 当前结论

本轮评审后，`02-user-guide/` 在当前项目范围内已经达到：

- 阅读路径明确
- 操作步骤完整
- 关键页面有成功标志
- 高频操作页有截图辅助

当前未发现阻断性的知识盲点。

## 5. 后续维护建议

这些不是当前阻断问题，但值得持续维护：

1. GitHub / PyPI 页面 UI 变化后，及时刷新 `publish-setup.md` 中的示意截图
2. 每次 CLI 对外行为变化后，至少复查：
   - `local-development.md`
   - `contributor-guide/cli.md`
   - `contributor-guide/release.md`
3. 下一次真实接入新源仓时，用新接入者做一次“冷读”验证，检查是否还能在 15 分钟内完成接入闭环
