# 项目章程

## 项目定义

`docs-stratego` 是一个单站点文档聚合与权限装配仓。目标是让多个项目继续在各自源仓维护 `docs/`，但在同一套导航中展示全部页面，并允许部分页面在点击后进入登录流程。

## 当前目标

- 为源仓定义统一的 `docs/` 标准和 `index.md` 元数据标准
- 在静态构建前生成导航配置和权限清单
- 使用开源认证服务支持“用户名密码 + GitHub 登录”
- 使用 Nginx 对静态页面做路径级访问控制
- 使用 Git submodule + sparse-checkout，仅展开子仓的 `docs/` 目录

## 边界原则

- 源仓自己维护页面内容、目录顺序和页面私有化声明
- 根仓只负责同步、校验、构建和部署
- 根仓不再改名文档、不再生成中间概览页、不再手写全站左侧导航
