# UI 与信息架构

## 浏览体验

- 全部页面都显示在同一套左侧导航里
- 顶部项目 Tab 首次点击应直接弹出下拉菜单，而不是先跳到项目首页
- 顶部项目 Tab 保持 Material 原有视觉风格，但可适度放大文字与字重，让入口更显眼
- 顶部项目 Tab 下拉菜单首项固定为“简介”，用于打开项目根 `index.md` 的简介内容
- 公开页面匿名可直接读
- 公开页面和私有页面继续共用同一套导航，不额外显示页面上的锁定标记
- 匿名用户点击私有页面时，先同步拉起独立的小窗登录框
- 登录链路在小窗内完成，通过同源桥接页用 `postMessage` 通知主页面
- 已登录状态下，私有页面应尽量走即时站内导航，避免因为重复鉴权检查导致白屏闪烁
- 浏览器如果不支持严格的小窗形态，允许退化为新标签页，但主页面不主动跳走
- 用户关闭登录小窗后，应留在当前公开页面继续浏览
- 用户点回主页面时，小窗应主动关闭，而不是仅退到后台
- 登录成功后回到原页面

## 信息架构原则

- 目录顺序由源仓 `index.md` 控制
- 页面标题直接来自 Markdown 一级标题
- 私有页面最好收敛到逻辑明确的目录中，避免资源权限混杂
