# 头脑风暴记录

## 背景

用户希望保留“一套目录、一套文档”的写作体验，同时做到：

- 左侧导航列出全部页面
- 私有页面点击后登录，登录后查看
- 页面私有化规则由源仓直接维护
- 根仓只获取源仓的 `docs/` 目录，避免拉取业务代码

## 方案比较

### 方案 A：双站点公开/私有分离

- 优点：隔离简单，安全边界清晰
- 缺点：一份文档被拆到两个站点，体验不符合当前目标

### 方案 B：单站点 + 静态构建 + 网关按路径登录

- 优点：符合“单站点、单导航、按页面登录”的目标
- 缺点：需要关闭或谨慎处理全站搜索索引，避免泄露私有内容

### 方案 C：动态应用型文档站

- 优点：可以做更细粒度的运行时权限
- 缺点：复杂度和运维成本显著增加，不适合当前阶段

## 结论

采用方案 B：

- 继续使用 MkDocs 生成静态页面
- 由根 `docs/index.md` 的全站清单生成导航和权限清单
- 使用 Casdoor 提供本地账号和 GitHub 登录
- 使用 oauth2-proxy 为 Nginx 提供标准 `auth_request` 接口
- 使用 Nginx 对私有页面 URL 做登录拦截
