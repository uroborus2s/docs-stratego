# PRD 摘要

- REQ-001：源仓必须通过统一 `docs/` 标准编写文档
- REQ-002：系统必须支持从多个源仓获取文档
- REQ-003：外部源仓同步时只获取 `docs/` 目录
- REQ-004：系统必须生成页面权限清单
- REQ-005：系统必须支持单站点展示全部页面，并对私有页面触发登录
- REQ-006：认证链路必须支持用户名密码和 GitHub 登录
- REQ-007：宿主机 Nginx 必须同时承接静态页面与认证代理入口
- REQ-008：系统必须保留可追溯的需求、方案、部署和运维文档
- REQ-009：子仓 `docs/**` 变更后必须自动通知根仓发起子仓指针同步
- REQ-010：根仓必须拆分“同步子仓指针”和“正式发布站点”的 workflow
- REQ-011：所有落后子仓指针必须汇总到同一个共享 bot PR
- REQ-012：共享 bot PR 在合并前必须经过自动校验和人工审核
- REQ-013：文档必须明确写出接入子仓自动联动所需的文件和凭证
